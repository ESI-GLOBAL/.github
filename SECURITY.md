# Security policy

Do not report vulnerabilities through public GitHub issues, pull requests, or discussions.

Use GitHub's private vulnerability reporting feature when it is available for the affected repository. If private reporting is unavailable, contact ESI Global through the security or support channel published on [esi-global.co](https://esi-global.co) and include:

- the affected public component or service;
- a concise description and potential impact;
- reproducible steps or a proof of concept that does not access other users' data;
- suggested remediation, if known;
- a safe way to contact you.

Do not include credentials, personal data, customer information, or exploit details in a public channel. ESI Global will acknowledge valid reports, assess severity, and coordinate remediation and disclosure where appropriate.
